# Contributing to Documentation

Thank you for contributing to our documentation!

## How to Contribute

1. Create a new branch for your changes
2. Add or update markdown files in the appropriate directory
3. Follow the style guidelines below
4. Submit a pull request

## Style Guidelines

### File Naming
- Use lowercase letters
- Use hyphens to separate words (e.g., `getting-started.md`)
- Use descriptive names that reflect the content

### Document Structure
- Start with a clear title (H1)
- Include a brief introduction
- Use appropriate heading levels (H2, H3, etc.)
- Add a table of contents for longer documents

### Markdown Best Practices
- Use code blocks with language specification for code examples
- Add alt text to images
- Use relative links for internal references
- Keep line lengths reasonable for readability

## Questions?

If you have questions about contributing, please open an issue in the repository.
