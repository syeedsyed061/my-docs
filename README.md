# My Documentation Repository

A repository for maintaining and organizing markdown documentation files.

## Repository Structure

```
my-docs/
├── README.md           # This file - repository overview
├── CONTRIBUTING.md     # Guidelines for contributing
├── docs/               # Main documentation files
│   └── getting-started.md
└── templates/          # Reusable markdown templates
    └── document-template.md
```

## Getting Started

1. Clone this repository
2. Add your markdown files to the `docs/` directory
3. Use templates from `templates/` as starting points for new documents

## Documentation Guidelines

- Use clear, descriptive filenames (e.g., `setup-guide.md`, `api-reference.md`)
- Follow the templates provided in the `templates/` folder
- Keep documentation up to date with code changes
- Use relative links when referencing other documents in this repository
